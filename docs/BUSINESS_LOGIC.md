# Expense Tracker: Огляд Проєкту та Дорожня Карта (Project Overview & Roadmap)

## 🎯 Головна ідея проєкту

Платформа для фінансового обліку та аналітики (Personal/Team Expense Tracker). Це сучасний вебдодаток, призначений для відстеження доходів і витрат, управління бюджетами за категоріями та надання детальної статистики. Проєкт створюється з акцентом на продуктивність, типізацію та ефективну роботу з базою даних.

### 🛠 Стек технологій (AI Context / Constraints)

ШІ-асистент ПОВИНЕН суворо дотримуватися цього стеку при генерації коду:

- **Framework:** Next.js (виключно App Router)
- **Language:** TypeScript (strict mode)
- **Styling:** Tailwind CSS
- **Data Fetching & State:** TanStack React Query (для клієнтського стану та мутацій)
- **Database:** PostgreSQL
- **Database Querying:** Kysely (Ніяких важких ORM на кшталт Prisma. Використовуємо типізований, raw-подібний SQL через Kysely).
- **Forms:** React Hook Form
- **Validation:** Zod
- **UI Library:** shadcn/ui

## 🚶‍♂️ User Journey (Як це працює загалом)

1. **Вхід та Налаштування:** Користувач заходить у додаток і потрапляє на головний дашборд. Спочатку він створює свої "Рахунки" (наприклад, "Готівка", "Банківська картка").
2. **Щоденне використання:** Користувач регулярно вносить свої транзакції (доходи або витрати) через зручну форму, вказуючи суму, дату, категорію та відповідний рахунок.
3. **Аналітика в реальному часі:** Після кожної транзакції система автоматично перераховує баланси рахунків.
4. **Репортинг:** На дашборді користувач бачить графіки, які показують, куди йдуть гроші, чи вписується він у встановлені бюджети, та порівняння витрат з попередніми періодами.

## 🚀 Етапи імплементації (Roadmap)

### Етап 1: Базовий фундамент (MVP - Minimal Viable Product)

**Мета:** Створити робочий інструмент для введення даних та перегляду списку транзакцій.

- [] **База даних (PostgreSQL + Kysely):** Спроєктувати та створити базові таблиці: `Users`, `Accounts` (Рахунки), `Categories` (Категорії), `Transactions` (Транзакції).
- [ ] **UI Форм:** Створити форму додавання транзакції, використовуючи `react-hook-form`, `zod` для валідації та компоненти `shadcn/ui` (Select, Input, DatePicker).
- [ ] **UI Відображення:** Реалізувати таблицю останніх операцій з базовою пагінацією.
- [ ] **Інтеграція:** Налаштувати зв'язок клієнтських форм із серверними екшенами (Server Actions) або API-роутами за допомогою мутацій `React Query`.

### Етап 2: Аналітика та UX (Core Value)

**Мета:** Зробити додаток швидким, інтерактивним та додати цінність у вигляді аналітики.

- [ ] **SQL Агрегації:** Написати складні запити через Kysely з використанням `SUM()`, `GROUP BY` для підрахунку витрат за поточний місяць у розрізі категорій.
- [ ] **Дашборд:** Додати візуалізацію даних (наприклад, інтеграція Recharts) — кругові діаграми розподілу витрат та стовпчасті діаграми динаміки по днях.
- [ ] **Optimistic Updates:** Налаштувати React Query так, щоб додавання/видалення транзакцій миттєво відображалося в UI без очікування відповіді від сервера.
- [ ] **Фільтрація:** Додати можливість фільтрувати транзакції за діапазоном дат та конкретним рахунком.

### Етап 3: Просунуті фічі (Scaling & Business Logic)

**Мета:** Ускладнити бізнес-логіку та підготувати проєкт до спільного використання.

- [ ] **Система Бюджетів:** Можливість встановити ліміт витрат на місяць для певної категорії. Індикація в UI (Progress Bar з `shadcn/ui`), коли ліміт вичерпується або перевищується.
- [ ] **Командні простори (Workspaces):** Створення сутності `Workspaces` для спільного ведення бюджету (наприклад, сімейний бюджет). Налаштування SQL-логіки перевірки прав доступу на рівні бекенду.
- [ ] **Експорт даних:** Генерація та завантаження звітів (CSV / PDF) за вибраний період.
